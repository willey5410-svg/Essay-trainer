# 英検1級 Essay Trainer

英検1級のエッセイ Body パラグラフを「固定テンプレート × スロット（ワイルドカード）」方式で構築・暗記するための Web アプリです。

## コンセプト

Body 1〜3 は次の4文構造（主張 → 原理 → 条件と帰結 → 結論）の固定テンプレートで、埋めるスロットは共通の6つです。

| スロット | 役割 | 例（AI規制） |
|---|---|---|
| Reason | 観点（動詞で考えて名詞化、5〜8語） | the large-scale reshaping of the labor force by AI |
| General principle | 原理（メカニズムまで述べる節、8〜13語） | AI handles complex judgment tasks at minimal cost |
| Condition | 条件 | companies automate without restraint |
| Result | 結果（主張と同じ方向・社会的な波及効果まで） | a reduced ability to adapt to new challenges |
| Key concept | キー概念 | unregulated automation |
| Conclusion | 結論 | destabilizing the labor market |

各 Body は例証文を省いた軽量テンプレート（固定 22 / 18 / 20 語）を使い、浮いた語数を観点・原理・結果の厚みに充て、組み立て後 45〜50 語程度に収まるよう生成されます。

**観点（Reason）生成の3原則**（論点出しトレーニングのヒントにも表示、Gemini への生成・判定プロンプトにも反映）：
1. **中立に書く**：賛成・反対の評価を含めず「何が変わるか」だけを書く（○ 労働力構造が再編される／× 批判的思考力が低下する）
2. **構造で考える**：「誰が得するか」ではなく、教育構造・労働力構造・情報流通・意思決定・社会制度・評価制度・市場構造・技術開発・資源配分など、どの構造が変わるかを探す
3. **一段抽象化する**：具体的な現象ではなく一段上の概念で表現する（「AIで宿題をする」→「学習プロセスが変化しうる」）

原理（なぜ起きるか）と結果（社会的な波及効果＝"So What?"）についても、この考え方に沿うよう生成時の指針に含めています。

思考の型：**A does B（動き）→ because（なぜ）→ leads to C（どう悪くなる／なぜ必要）**

## 使い方

1. **初回起動時に合言葉を入力** — サーバー側で検証され、この端末に保存されます（スキップするとサンプル練習のみ利用可）
2. **テーマを選ぶ** — カテゴリ別のテーマ案から選択。テーマは自分で追加（英語・和訳・カテゴリ）でき、×ボタンで削除も可能（プリセットは非表示化され、ワンタップで復元できます）。Gemini によるテーマ案の追加生成にも対応
3. **スタンスを選ぶ** — 賛成（YES）／反対（NO）
4. **論点出しトレーニング** — 生成の前に90秒のタイマー付きで自分の論点を3つ入力（A does B の形、日本語可）。Gemini が各論点の有効性を判定し、生成された論点と並べて比較表示（スキップも可能）。**学習画面の「🧠 論点だしトレーニングをもう一度」からは、生成済みのエッセイに対して何度でも反復練習ができます**（エッセイ本文は変更されず、最新1回分の判定結果だけが保存されます。プリセットのサンプルエッセイでも利用可能）
5. **Gemini が例文を生成** — テンプレートに沿って Body 1〜3 のスロット値＋和訳を単発のリクエストで生成（各 Body 約45〜50語）
6. **Gemini が試験官として採点** — 生成完了後、バックグラウンドの別リクエストで構成・内容・英語表現の3観点（各10点）を自己採点し、完了次第スコアと日本語講評を表示。既存のエッセイに対しても「この構成を採点する」からいつでも実行可能。スコアが低い場合や単に別パターンを試したい場合は「🔄 別パターンで再生成」で同じテーマ・立場から作り直せます
7. **練習する** — テンプレートのワイルドカード部分が空欄で表示され、シャッフルされた単語チップを正しい順にタップして文章を組み立てる
8. **自分の言葉で書き換える** — 練習中の空欄（「✍️ 自分の言葉で」）や学習画面のスロット（タップ）から自由に英文を入力すると、Gemini が適否を判定（✅問題なし／△修正あり／✖不適合）し、添削文と日本語解説・更新された和訳を提示。ワンタップで模範解答を自分の表現に置き換えられ、「元の模範解答に戻す」も可能（編集すると採点はやり直しになります）
9. **論点でBodyを書き換える** — 論点だしトレーニング（反復練習）で作った論点のうち、有効と判定されたもの（✅／△）には「→ Bodyに反映」ボタンが表示されます。押して Body 1〜3 のどれを書き換えるか選ぶと、その論点を核に6スロットすべてを Gemini が書き直します（プレビュー後に適用／取り消し可能）
10. **Geminiに質問する** — 学習画面の「💬 Geminiに質問する」から、採点結果や論点判定について自由に会話できます（例：「なぜ内容のスコアが低いのですか？」）。会話はエッセイごとに保存され、「🗑 会話をリセット」でいつでもクリア可能
11. **進捗を確認** — Body ごとの実施回数・ベスト正答率を記録

**設計上の制約**：1回の操作（生成・採点・テーマ提案・添削）は必ず Gemini API 呼び出し1回で完結します。生成と採点をあえて別リクエストに分離しているのは、Vercel の無料（Hobby）プランではサーバーレス関数の実行時間に上限があり、複数のAPI呼び出しを1リクエスト内で連鎖させると、その上限を超えて「サーバーに接続できません」というエラーになることがあるためです。

合言葉なしでも、同梱のサンプルエッセイ2本（AI規制・大学無償化）で練習できます。

設定画面から**テーマ（配色）を「自動／ライト／ダーク」から選択**できます。「自動」は端末（OS）のダークモード設定に追従します。

## アーキテクチャとセキュリティ

Gemini API キーは **ブラウザには一切渡されません**。ブラウザは Vercel サーバーレス関数 `/api/generate` を呼び出し、関数がサーバー側の環境変数からキーを読んで Gemini API に中継します。関数の利用には環境変数 `APP_KEYWORD` と一致する合言葉が必要です。

### データ保存（Vercel Blob 同期）

学習データ（生成したエッセイ・進捗・テーマ）は **Vercel Blob をマスター、localStorage をキャッシュ兼オフライン用**として保存します。

- 変更は自動でクラウドにアップロード（2.5秒のデバウンス付き）。ホーム画面右上のバッジ（☁ 同期済み／同期中…）で状態を確認できます
- 起動時にクラウドから取得するため、**別の端末でも合言葉を入れるだけで同じデータが使えます**
- Blob は **Private アクセス**で保存され、読み書きはサーバー側のトークンで認証された `/api/data` 経由でのみ行われます（公開URLからは読めません）。Blob のパスも合言葉から導出したハッシュを含みます（合言葉を変更すると新しいデータ領域になります）
- Blob ストア未設定の環境では自動的に「💾 ローカル保存」モードで動作します。設定画面から JSON エクスポート／インポートも引き続き可能です

## セットアップ（Vercel）

1. このリポジトリを Vercel に Import（設定変更は不要。`api/` が自動でサーバーレス関数として認識されます）
2. Vercel のプロジェクト設定 → **Environment Variables** に以下を登録して再デプロイ：

| 変数名 | 内容 |
|---|---|
| `GEMINI_API_KEY` | [Google AI Studio](https://aistudio.google.com/apikey) で取得した API キー（必須） |
| `APP_KEYWORD` | アプリ利用時の合言葉（必須・自由な文字列） |
| `GEMINI_MODEL` | 使用モデル（任意。省略時 `gemini-2.5-flash`） |
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob のトークン。**手動登録は不要** — Vercel の Storage タブから Blob ストアを作成しプロジェクトに接続すると自動で設定されます |

3. **クラウド同期を有効にする**：Vercel ダッシュボード → プロジェクト → **Storage** タブ → **Create Database** → **Blob** を作成してプロジェクトに Connect → Redeploy。これだけで学習データが Blob に保存されるようになります（未設定でもローカル保存で動作します）

### ローカル開発

サーバーレス関数を含めて動かすには Vercel CLI を使います：

```bash
npx vercel dev
```

（`GEMINI_API_KEY` と `APP_KEYWORD` を `.env` または環境変数で指定）

## ファイル構成

```
index.html        エントリポイント
css/style.css     スタイル
js/templates.js   Body 1〜3 テンプレート定義（固定テキスト＋スロット）
js/presets.js     プリセットテーマとサンプルエッセイ
js/gemini.js      /api/generate 呼び出し（合言葉付き）
js/app.js         画面・練習ロジック・進捗管理
api/generate.js   Vercel サーバーレス関数（Gemini プロキシ＋合言葉検証）
```
